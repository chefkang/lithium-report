### 创建自动刷新脚本
因GitHub Pages CDN缓存刷新缓慢，创建了多种自动刷新方案：

#### 1. GitHub Actions 工作流
- **文件**: `.github/workflows/refresh-cache.yml`
- **功能**: 每次推送到master自动触发，更新时间戳并刷新缓存
- **状态**: ✅ 已创建，等待推送后生效

#### 2. 本地刷新脚本
- **Windows批处理**: `refresh-github-pages.bat`
- **Bash脚本**: `refresh-github-pages.sh`
- **功能**: 本地运行，通过GitHub API触发Pages重建
- **使用方法**: 需要提供GitHub Personal Access Token

#### 3. Git忽略配置
- **文件**: `.gitignore`
- **作用**: 排除敏感文件（Token、本地状态等）

  - ✅ 已配置Hosts文件加速（临时缓解）

### Git代理配置命令
```bash
# 配置代理（以Clash默认端口7890为例）
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890

# 取消代理
git config --global --unset http.proxy
git config --global --unset https.proxy
```
