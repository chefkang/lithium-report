# 上下文管理策略 - 减少 Token 浪费

## 问题
- 长对话历史导致每次重启后需要大量 token 来恢复上下文
- 没有有效的「断点续传」机制

## 解决方案

### 1. 消息数阈值提醒
当对话超过 70 条消息时，主动提醒用户保存上下文并开启新 session。

### 2. 会话中断前的「断点保存」
在以下情况发生时，自动保存状态：
- 完成一个重要任务
- 对话超过 50 条消息
- 用户说"明天继续"、"先这样"等结束语

### 3. 断点保存格式
保存文件：`memory/checkpoint_项目名称_日期.md`

内容模板：
```markdown
# Checkpoint: 项目名称
## 日期: YYYY-MM-DD HH:MM

## 当前状态
- 完成了: xxx
- 正在做: xxx
- 下一步: xxx

## 关键代码/配置
```

## 关键文件位置
- `memory/README.md` - 记忆系统说明
- `memory/YYYY-MM-DD.md` - 每日记录
- `memory/checkpoint_*.md` - 项目断点
- `MEMORY.md` - 长期记忆（重要）

## 执行规则
- **我必须在每天结束时主动创建/更新记忆文件**
- **我必须在重要节点创建 checkpoint**
- **我必须在新会话开始时读取相关记忆文件**
